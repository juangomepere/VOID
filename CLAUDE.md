## What is VOID

VOID is a two-player competitive maze game built with vanilla HTML5 Canvas. Both players race to collect clocks that extend their countdown timer — last one alive wins. The entire game lives in a single file: `index.html`.

## Running the Game

No build step. Open `index.html` directly in a browser, or serve it:

```
python -m http.server 8000
# or
npx http-server .
```

No package.json, no dependencies, no compilation.

## Code Architecture

All ~437 lines of game logic are in `index.html` as inline JavaScript. There are no modules or imports.

**Key global state:**
- `maze` — 25×25 grid of wall objects `{n, s, e, w}` generated by recursive DFS
- `P` — two player objects: `{x, y, time, score, alive, color, controls}`
- `clocks` — active collectible objects on the grid; types: bronze (+20s), silver (+45s), gold (+90s), shadow (−40s)
- `footprints`, `echos`, `walls` — transient visual effect arrays

**Game loop** (`requestAnimationFrame` at 60fps):
1. `update(dt)` — advances timers, checks clock pickups, spawns new clocks, ends game
2. `draw()` — renders maze, clocks, players, effects to a 600×600px canvas (24px cells)

**Phase progression** — `getPhase()` returns one of 4 phases (Ciega → Pulso → Sombra → Noche) based on elapsed time, which controls clock spawn rate and visual style.

**Controls:**
- P1: WASD
- P2: Arrow keys

Movement is validated against maze wall data; collisions trigger a brief wall-flash effect.

## Making Changes

Since everything is in one file with global state, keep changes isolated to the logical section they belong to:
- Maze generation: top of the JS block
- Game state init / reset: `initGame()`
- Per-frame logic: `update(dt)`
- Rendering: `draw()` and its helper functions
- Input: the `keydown` listener

The canvas is 600×600px; the grid is 25×25 cells at 24px each — math involving positions must stay consistent with these constants.
